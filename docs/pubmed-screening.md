# PubMed 筛选逻辑

## 业务问题

PubMed 中大量文章会因为标题或摘要出现 `aging`、`older adults` 等词而命中，但其中许多研究关注的是老年群体的常规疾病管理，并没有讨论衰老机制或健康寿命。

因此，工作流不采用“关键词命中即发布”，而是将检索、去重、来源分流和语义筛选拆开。

## 两条检索通道

### 优先期刊通道

对重点期刊使用独立检索，以较小的结果窗口保证及时处理。当前公开版本包含 NEJM、Ageing Research Reviews 与 PNAS。

### 广域发现通道

在其他 PubMed 期刊中使用衰老与长寿相关关键词进行发现，同时排除已经被 CNS 或其他独立管线覆盖的来源，避免重复抓取和重复发布。

## 去重与排除

- 使用 PMID 作为稳定标识
- 在 n8n static data 中保留最近处理的 PMID
- 对来源名称执行二次黑名单匹配，处理 PubMed 期刊缩写与名称差异
- 检索窗口限定最近一天，减少重复和历史噪声

## 语义守门

代码规则负责确定性排除，模型负责判断文章的研究核心。

通过方向包括：

- 细胞衰老与 senescence
- 端粒与表观遗传时钟
- 线粒体衰退与干细胞耗竭
- mTOR、AMPK、Sirtuins、NAD+、IGF-1
- 自噬、蛋白质稳态与 DNA 损伤修复
- 免疫衰老与肠道菌群的衰老机制
- 直接影响寿命或健康寿命的干预

排除方向包括：

- 纯护理、生活质量和慢性病管理
- 只有年龄特征、没有衰老机制的临床研究
- 纯社会心理或行为学研究
- 纯患病率、发病率和相关性统计
- 一般药物安全性、药代和常规疗效研究

## 为什么不让模型承担全部逻辑

- 日期、期刊、PMID 和重复判断适合代码
- 研究核心和机制贡献适合语义模型
- 分工后更容易逐节点调试，也更容易解释为什么文章被纳入或排除
